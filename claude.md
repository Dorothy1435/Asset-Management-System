# CLAUDE.md

## 프로젝트 개요

이 프로젝트는 조직 내 자산을 등록, 관리, 검색할 수 있는 **자산관리 웹페이지**이다.

사용자는 자산의 위치, 대여자, 물품 사진, 등록자 정보를 직접 등록할 수 있어야 하며, 등록된 자산은 검색 기능을 통해 쉽게 조회할 수 있어야 한다.

최종 목표는 엑셀이나 수기 관리 대신 웹페이지에서 자산 정보를 한눈에 확인하고 관리할 수 있도록 하는 것이다.

---

## 주요 기능

### 1. 자산 등록 기능

사용자는 새로운 자산 정보를 등록할 수 있어야 한다.

등록 항목은 다음과 같다.

* 자산명
* 자산번호
* 자산 위치
* 대여자
* 물품 사진
* 등록자
* 등록일
* 비고

각 항목은 입력 폼으로 구성한다.

필수 입력 항목은 다음과 같다.

* 자산명
* 자산번호
* 자산 위치
* 등록자

대여자는 자산이 대여 중일 경우 입력하고, 대여 중이 아니면 비워둘 수 있도록 한다.

물품 사진은 이미지 파일을 업로드할 수 있도록 한다.

---

### 2. 자산 목록 조회 기능

등록된 자산은 목록 형태로 확인할 수 있어야 한다.

목록에는 기본적으로 다음 정보가 보이도록 한다.

* 자산명
* 자산번호
* 자산 위치
* 대여자
* 등록자
* 등록일
* 상태

상태는 다음과 같이 구분한다.

* 보관 중
* 대여 중

대여자 정보가 있으면 “대여 중”으로 표시하고, 대여자 정보가 없으면 “보관 중”으로 표시한다.

---

### 3. 자산 검색 기능

사용자는 등록된 자산을 검색할 수 있어야 한다.

검색 가능한 항목은 다음과 같다.

* 자산명
* 자산번호
* 자산 위치
* 대여자
* 등록자

검색창에 키워드를 입력하면 해당 키워드가 포함된 자산 목록이 나오도록 한다.

예를 들어 “A동”을 검색하면 자산 위치가 A동인 물품들이 나오고, “홍길동”을 검색하면 대여자 또는 등록자가 홍길동인 자산들이 나오도록 한다.

---

### 4. 자산 상세 보기 기능

목록에서 자산을 클릭하면 상세 정보를 확인할 수 있어야 한다.

상세 화면에는 다음 정보가 표시되어야 한다.

* 물품 사진
* 자산명
* 자산번호
* 자산 위치
* 대여자
* 등록자
* 등록일
* 상태
* 비고

사진이 등록되지 않은 경우에는 “등록된 사진 없음”이라고 표시한다.

---

### 5. 자산 수정 기능

등록된 자산 정보는 수정할 수 있어야 한다.

수정 가능한 항목은 다음과 같다.

* 자산명
* 자산번호
* 자산 위치
* 대여자
* 물품 사진
* 등록자
* 비고

자산 위치가 바뀌거나 대여자가 바뀌는 경우 쉽게 수정할 수 있도록 한다.

---

### 6. 자산 삭제 기능

필요 없는 자산 정보는 삭제할 수 있어야 한다.

삭제 버튼을 누르면 바로 삭제하지 말고, 확인 메시지를 띄운다.

예시 문구:

“정말 이 자산을 삭제하시겠습니까?”

사용자가 확인을 누른 경우에만 삭제한다.

## 화면 구성

메인 화면에는 다음 요소를 배치한다.

* 페이지 제목: 자산관리 시스템
* 자산 등록 버튼
* 검색창
* 자산 목록 테이블

검색창은 화면 상단에 배치하고, 자산 목록은 그 아래에 표시한다.

---

### 2. 자산 등록 화면

자산 등록 화면에는 입력 폼을 배치한다.

입력 폼 항목은 다음과 같다.

* 자산명
* 자산번호
* 자산 위치
* 대여자
* 물품 사진 업로드

* 등록
* 취소

### 3. 자산 상세 화면

자산 상세 화면에는 선택한 자산의 모든 정보를 보여준다.

상세 화면에는 다음 버튼을 배치한다.

* 수정
* 삭제
* 목록으로 돌아가기

## 데이터 구조

자산 데이터는 다음 구조를 기준으로 한다.

```js
{
  id: "고유 ID",
  assetName: "자산명",
  assetNumber: "자산번호",
  location: "자산 위치",
  borrower: "대여자",
  imageUrl: "물품 사진 경로",
  registrant: "등록자",
  registeredDate: "등록일",
  status: "보관 중 또는 대여 중",
  note: "비고"
}
```

status 값은 borrower 값이 있는지 없는지에 따라 자동으로 결정한다.

borrower 값이 있으면 status는 “대여 중”으로 한다.

borrower 값이 없으면 status는 “보관 중”으로 한다.

## 디자인 요구사항

전체 디자인은 깔끔하고 단순하게 만든다.

관리자가 빠르게 자산을 확인할 수 있도록 표 형태를 중심으로 구성한다.

화면은 다음 느낌으로 만든다.

* 공공기관 또는 대학 행정 시스템 느낌
* 흰색 배경
* 연한 회색 테두리
* 파란색 계열 버튼
* 읽기 쉬운 글자 크기
* 복잡하지 않은 화면 구성

## 기능 구현 세부사항

### 등록

사용자가 입력한 정보를 받아 자산 데이터를 생성한다.

자산 등록 시 id는 자동 생성한다.

등록일은 현재 날짜로 자동 저장한다.

자산번호가 중복되지 않도록 검사한다.

이미 등록된 자산번호를 입력하면 다음 메시지를 보여준다.

“이미 등록된 자산번호입니다.”

---

### 검색

검색창에 입력된 키워드를 기준으로 자산 목록을 필터링한다.

검색 대상은 다음 필드이다.

* 자산명
* 자산번호
* 위치
* 대여자
* 등록자

대소문자는 구분하지 않는다.

검색어가 비어 있으면 전체 자산 목록을 보여준다.

---

### 수정

수정 버튼을 누르면 기존 자산 정보가 입력 폼에 채워진 상태로 표시된다.

사용자가 내용을 수정하고 저장하면 기존 데이터가 변경된다.

수정 후에는 자산 목록 화면으로 돌아간다.

---

### 삭제

삭제 버튼을 누르면 확인창을 띄운다.

사용자가 확인하면 해당 자산을 삭제한다.

삭제 후에는 목록을 다시 새로고침한다.

---

### 사진 업로드

물품 사진은 이미지 파일만 업로드할 수 있도록 한다.

업로드한 이미지는 미리보기가 가능하도록 한다.

localStorage를 사용할 경우 이미지는 Base64 형태로 저장할 수 있다.

사진이 없을 경우 기본 이미지 영역 또는 “사진 없음” 문구를 표시한다.

## 예외 처리

다음 상황에 대한 안내 메시지를 보여준다.

* 필수 입력값이 비어 있는 경우
* 자산번호가 중복된 경우
* 검색 결과가 없는 경우
* 사진 파일이 이미지 형식이 아닌 경우

예시 문구:

* “필수 항목을 입력해주세요.”
* “이미 등록된 자산번호입니다.”
* “검색 결과가 없습니다.”
* “이미지 파일만 업로드할 수 있습니다.”

## 개발 우선순위

1단계에서는 다음 기능을 먼저 구현한다.

* 자산 등록
* 자산 목록 조회
* 자산 검색
* 자산 상세 보기

2단계에서는 다음 기능을 추가한다.

* 자산 수정
* 자산 삭제
* 사진 업로드 및 미리보기

3단계에서는 다음 기능을 개선한다.

* 디자인 정리
* 반응형 화면
* 상태 배지 표시
* 검색 필터 고도화

---

## 최종 결과물 목표

최종 결과물은 다음 조건을 만족해야 한다.

* 사용자가 자산 정보를 직접 등록할 수 있다.
* 등록된 자산 목록을 한눈에 볼 수 있다.
* 자산명, 자산번호, 위치, 대여자, 등록자로 검색할 수 있다.
* 자산 상세 정보를 확인할 수 있다.
* 자산 정보를 수정하거나 삭제할 수 있다.
* 물품 사진을 등록하고 확인할 수 있다.
* 전체 화면은 깔끔하고 행정 업무용 시스템처럼 보여야 한다.
